# Task5 项目规划

## 1. 项目定位

《校园轻集市》是一个面向校园场景的轻量级二手交易与信息发布平台。前端项目需要支持学生浏览商品、搜索筛选、查看详情、发布商品，并能完成基础的个人中心与商品管理操作。

当前仓库是 Vue 3 种子项目，已经具备入口文件、路由、状态管理和工程化配置，后续开发将在此基础上逐步补充业务页面和功能模块。

## 2. 页面清单

| 页面 | 路由建议 | 主要内容 |
| --- | --- | --- |
| 首页 | `/` | 项目首页、推荐商品、分类入口、搜索入口 |
| 商品列表页 | `/products` | 商品卡片列表、分类筛选、关键词搜索、排序 |
| 商品详情页 | `/products/:id` | 商品图片、标题、价格、描述、发布者信息、联系入口 |
| 发布商品页 | `/publish` | 商品标题、分类、价格、描述、图片上传表单 |
| 登录页 | `/login` | 学号或账号登录表单 |
| 注册页 | `/register` | 基础用户注册信息 |
| 个人中心 | `/profile` | 用户信息、我的发布、我的收藏 |
| 我的发布 | `/my-products` | 查看、编辑、下架自己发布的商品 |
| 收藏列表 | `/favorites` | 查看已收藏商品 |
| 404 页面 | `/:pathMatch(.*)*` | 处理不存在的路由 |

## 3. 功能模块

### 3.1 商品模块

- 商品列表展示；
- 商品分类筛选；
- 商品关键词搜索；
- 商品详情展示；
- 商品发布、编辑和下架；
- 商品状态区分，例如在售、已售、下架。

### 3.2 用户模块

- 用户登录；
- 用户注册；
- 用户信息展示；
- 登录状态维护；
- 退出登录。

### 3.3 发布模块

- 发布商品表单；
- 表单必填校验；
- 商品价格、分类、描述填写；
- 图片上传或图片占位处理；
- 发布成功后的页面跳转。

### 3.4 收藏模块

- 收藏商品；
- 取消收藏；
- 查看收藏列表。

### 3.5 路由与状态模块

- 使用 Vue Router 管理页面跳转；
- 使用 Pinia 管理登录用户、商品数据、收藏数据；
- 对需要登录的页面增加路由守卫，例如发布页和个人中心。

### 3.6 文档与过程证据模块

- 持续维护 AI 协作记录；
- 每日填写 Evidence；
- 保持开发过程可追踪。

## 4. 开发顺序

1. 熟悉项目结构，确认 `main.ts`、`App.vue`、`router` 的关系。
2. 规划页面路由，先补齐首页、商品列表页、详情页、发布页、个人中心等基础路由。
3. 设计基础数据结构，例如商品、用户、分类、收藏。
4. 实现首页和商品列表页，先让用户能看到商品信息。
5. 实现商品详情页，完成从列表到详情的跳转。
6. 实现发布商品页，完成表单输入和基础校验。
7. 引入 Pinia store，集中管理商品数据、用户状态和收藏状态。
8. 实现登录、注册、个人中心等用户相关页面。
9. 完善收藏、我的发布、编辑和下架等扩展功能。
10. 统一页面样式，检查移动端和桌面端的基础显示效果。
11. 运行类型检查、构建和代码规范检查。
12. 整理 AI 协作记录、Evidence 和最终项目说明。

## 5. 开发重点

我认为本项目的开发重点包括以下几项：

- 路由设计要清晰，页面之间的跳转关系要符合用户使用习惯。
- 商品数据结构要提前规划好，否则后面列表、详情、发布和个人中心都容易重复修改。
- 发布商品表单要重视校验，例如标题、价格、分类、描述不能为空。
- 状态管理要适度使用 Pinia，把跨页面共享的数据放到 store 中，不把所有逻辑都堆在单个组件里。
- 页面组件要尽量复用，例如商品卡片、分类筛选、空状态提示、表单输入区域。
- 文档和 Evidence 要真实记录，不只写结果，也要记录理解过程、遇到的问题和 AI 协作情况。

## 6. 阶段性目标

| 阶段 | 目标 |
| --- | --- |
| 第一阶段 | 完成项目结构分析、技术栈理解、项目规划和 Day1 Evidence |
| 第二阶段 | 完成基础路由和主要页面框架 |
| 第三阶段 | 完成商品列表、详情、发布等核心功能 |
| 第四阶段 | 完成用户状态、个人中心、收藏等功能 |
| 第五阶段 | 完成样式优化、测试检查和文档整理 |

## 7. 当前结论

当前项目已经具备前端工程基础，下一步不应急于写复杂功能，而应该先完成页面和数据结构设计。等页面清单、路由结构和核心模块明确后，再逐步实现商品、用户、发布和收藏功能，可以减少后期返工。
