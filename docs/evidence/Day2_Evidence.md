## 2. 学习收获

通过今天的开发，我进一步理解了 Vue 单页应用中“页面组件、路由、布局组件”之间的关系。

页面文件本身只是 Vue 组件，只有在 `src/router/index.ts` 中配置路由后，用户才能通过浏览器地址或导航菜单访问对应页面。`router-view` 负责展示当前路由匹配到的页面组件，公共布局组件则可以让不同页面共享统一的顶部栏、导航和页面容器。

我也认识到，页面结构应该优先围绕业务分类来设计。本次把校园轻集市拆分为二手交易、失物招领、拼单搭子、跑腿委托、发布、消息和个人中心，可以让后续功能继续按模块扩展。

## 3. 遇到的问题

今天遇到的主要问题如下：

- 原有部分页面和文档存在中文乱码，需要重新整理为可读的中文内容。
- 原有路由是 `/home`、`/list`、`/profile` 等通用路径，和 Day2 要求的业务路径不一致，因此需要重新配置。
- 项目中最初没有 `src/assets/main.css`，需要补充全局样式文件并在 `main.ts` 中引入。
