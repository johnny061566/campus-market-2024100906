# Day5 Evidence - Pinia 全局状态管理

## 1. 今日完成内容

今天主要完成了 Pinia 全局状态管理接入。项目原本已经在 `main.ts` 中挂载了 Pinia，本次在此基础上新增了用户状态 Store 和收藏状态 Store，并把它们应用到导航栏、发布页、四类业务列表页和个人中心页面中。

用户信息现在统一来自 `userStore`，发布信息时不再写死发布人。收藏状态统一来自 `favoriteStore`，列表页可以收藏或取消收藏，个人中心可以查看当前收藏列表。

## 2. Store 设计

| Store 文件 | 主要作用 | 当前状态 |
|---|---|---|
| `src/stores/user.ts` | 管理当前用户信息 | 登录状态、用户姓名、学院、年级、头像、简介 |
| `src/stores/favorite.ts` | 管理收藏列表 | 收藏类型、id、标题、描述、标签、地点 |

本次没有把所有接口列表都放入 Store，只保存多个页面需要共享的状态，符合 Day5 “共享状态才放入 Store” 的原则。

## 3. 页面接入情况

- `AppHeader.vue`：显示当前用户姓名、学院和年级；
- `AppNav.vue`：在个人中心入口显示收藏数量；
- `PublishView.vue`：发布人和联系人改为读取 `userStore.displayName`；
- `TradeView.vue`：二手交易卡片支持收藏和取消收藏；
- `LostFoundView.vue`：失物招领表格支持收藏和取消收藏；
- `GroupBuyView.vue`：拼单搭子卡片支持收藏和取消收藏；
- `ErrandView.vue`：跑腿委托列表支持收藏和取消收藏；
- `UserCenterView.vue`：展示用户资料、我的发布基础结构和我的收藏列表。

## 4. 我的发布实现思路

Day5 阶段先实现“我的发布”的基础结构展示。发布页已经把新内容的发布人设置为当前用户姓名，后续如果需要继续完善，可以在个人中心中请求四类业务数据，再根据 `publisher` 或 `contact` 字段筛选当前用户发布的内容。

## 5. 收藏功能说明

收藏状态暂时保存在前端内存中，刷新页面后会丢失。这符合 Day5 任务要求，当前重点是理解 Pinia 在多个页面之间共享状态的用法。后续如果需要持久化，可以继续结合 `localStorage` 或后端接口。

## 7. 遇到的问题与解决方法

四类业务数据结构并不完全相同，例如二手交易、拼单搭子和跑腿委托使用 `publisher`，失物招领使用 `contact`。因此收藏 Store 没有直接保存完整业务对象，而是保存统一的收藏展示结构：类型、id、标题、描述、标签和地点。这样个人中心可以用同一套 UI 展示不同来源的收藏内容。

另一个问题是终端默认环境中没有可用的 Node 命令。解决方法是使用 Codex 工作区依赖中提供的 Node 和 pnpm 路径完成验证。

## 8. 今日反思

Pinia 的作用不是替代页面自己的局部状态，而是管理多个页面或多个组件都需要访问的共享状态。通过本次任务可以看到，当前用户信息和收藏列表适合放入 Store，而表单输入、校验错误和单个页面的列表加载状态仍然保留在页面组件中更清晰。
